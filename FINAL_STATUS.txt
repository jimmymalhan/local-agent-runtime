WHY THIS WASN'T AUTOMATED BEFORE
═════════════════════════════════════════════════════════════════════════════════

1. ARCHITECTURAL GAP
   ───────────────────
   The system had 3 independent components built at different times:
   • Layer 1: Projects Queue (projects.json, ProjectManager)
   • Layer 2: Task Executor (task_suite.py, orchestrator/main.py)
   • Layer 3: Infrastructure (health checks, startup scripts)

   They were NEVER WIRED TOGETHER.
   Result: System running but executing WRONG TASKS.

2. BENCHMARK-FIRST DESIGN
   ──────────────────────
   The orchestrator was designed for continuous self-improvement:
   • Compare against Opus 4.6
   • Track improvements across versions (v1→v1000)
   • Auto-upgrade prompts
   
   Real project execution was added LATER but never integrated.

3. NO AUTOMATIC VERIFICATION
   ──────────────────────────
   Nobody asked: "Are we doing what we're supposed to do?"
   
   A simple 30-minute check would have caught it:
   if projects_completed == 0:
       CRITICAL("Task queue not being executed")

4. ASSUMPTION OF HUMAN OPERATOR
   ────────────────────────────
   The original design assumed:
   • Manual startup
   • Manual monitoring
   • Manual restarts
   • Manual verification

   This assumption was never documented or questioned.

5. LACK OF PERMANENT AUTO-STARTUP
   ──────────────────────────────
   No launchd/systemd service = system dies on crash and stays dead.
   
   Manual workarounds:
   • Cron job: * * * * * bash startup.sh (runs every minute, wasteful)
   • Ignore: Accept crashes and hope someone notices

FULL EXPLANATION: See WHY_SYSTEMS_WERENT_AUTOMATED.md


SETUP: PERMANENT AUTO-STARTUP (macOS)
═════════════════════════════════════════════════════════════════════════════════

Install once (requires sudo):

  sudo cp scripts/com.local-agents.system.plist \
    /Library/LaunchDaemons/com.local-agents.system.plist

  sudo launchctl load /Library/LaunchDaemons/com.local-agents.system.plist

Verify:
  launchctl list | grep local-agents

View logs:
  tail -f local-agents/logs/launchd_daemon.log

Uninstall:
  sudo launchctl unload /Library/LaunchDaemons/com.local-agents.system.plist
  sudo rm /Library/LaunchDaemons/com.local-agents.system.plist
